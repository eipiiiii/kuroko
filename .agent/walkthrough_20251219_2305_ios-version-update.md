# iOSバージョン対応変更 Walkthrough

## 概要
ユーザーの「ios26のみ対応に変更できる？」という問い合わせに対し、プロジェクトのドキュメントをiOS 26.0対応に更新しました。

## 変更内容

### 確認された事実
- XcodeプロジェクトのIPHONEOS_DEPLOYMENT_TARGETは既に26.0に設定されている
- ドキュメント（rules.md, README.md）ではiOS 17+と記載されており、不整合があった

### 実施した変更
1. **rules.md更新**
   - 「Swiftバージョン: iOS 17.0+ / macOS 14.0+」を「iOS 26.0+ / macOS 26.0+」に変更

2. **README.md更新**
   - バッジのiOSバージョンを「17+」から「26+」に変更
   - 必要条件のiOSバージョンを「17+」から「26+」に変更

### 技術的詳細
- プロジェクト設定（project.pbxproj）は既にiOS 26.0をターゲットにしているため、変更不要
- ドキュメントの整合性を確保するため、rules.mdとREADME.mdのみ更新

## 結果
- プロジェクトのドキュメントがiOS 26.0対応に統一された
- Xcodeプロジェクト設定とドキュメントの間に矛盾が解消された

## 注意点
- iOS 26はまだリリースされていないため、実際のビルド・テストは将来のXcode/iOS SDKリリースまで待つ必要がある
- 現在の環境（2025年12月19日）ではiOS 26.0ターゲットでの開発は可能だが、シミュレータや実機での動作確認は限定的

- 新しいAPIの利用可能性を確認
